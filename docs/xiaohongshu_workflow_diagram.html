小红书团队完整工作流程图

适用于 ClawX + OpenClaw + OpenProse + Chrome Relay 的“老板调度 / 员工执行 / 人工检查接管”模式

1. 用户输入层

你在 ClawX 中输入自然语言任务，例如：采集 2026 服装、近 30 天、按点赞前 20、补评论区话题、导出到桌面。

↓

2. xiaohongshu-boss 指令理解层

  • 理解关键词 / 主题
  • 识别时间窗、排序规则、数量
  • 识别是否需要评论区、主页、趋势等细节层
  • 判断输出格式和命名规则
  • 预判是否会触发登录、人机验证、验证码
↓

3. Mission 结构化

  • keyword_or_subject
  • time_window
  • ranking_rule
  • target_count
  • detail_layer
  • output_path
  • filename_prefix
  • validation_checks

4. OpenProse 工作流引擎

  • 把自然语言任务变成可执行流程
  • 按阶段派工，而不是让老板亲自做
  • 支持暂停、继续、重试、验收
↓

5. 派工阶段

  • 把小红书任务分给专项员工
  • 把网页动作分给浏览器员工
  • 把导出与文件落地分给本地文件员工
↓

6. 人工 Checkpoint 判断

  • 是否登录
  • 是否出现滑块 / 验证码 / Google 人类检查
  • 是否出现 2FA 或强制身份确认
↓

7. 暂停与恢复

如果遇到验证，流程暂停，交给你手动完成。你处理完成后，系统从断点继续，而不是整轮重来。

8. xiaohongshu-suite 专项员工

  • 搜索关键词
  • 采集可见笔记卡片
  • 提取标题 / 作者 / 点赞 / 链接
  • 点赞归一化
  • Top N 排序
  • 详情补采
  • 评论区话题关键词提取

9. browser-assist 浏览器员工

  • 接管 Chrome 当前登录会话
  • 打开搜索页
  • 点击笔记卡片进入详情
  • 滚动页面加载评论
  • 遇到验证时暂停并等待人工处理

10. windows-ops 本地文件员工

  • 生成 CSV
  • 生成 Markdown 总结
  • 保存到桌面
  • 自动按关键词命名文件
  • 例如：2026服装_top20.csv

11. boss-monitor 监工层

  • 检查有没有跑错关键词
  • 检查排序规则是否一致
  • 检查数量是否满足
  • 检查评论细节有没有缺失
  • 检查文件名前缀是否匹配任务主题
  • 检查是否抓到了错误页面
↓

12. validator 验收层

  • 核对输出是否符合原始命令
  • 不通过则回退到对应员工重跑
  • 通过才允许老板汇报完成
↓

13. 最终交付层

  • CSV 表格
  • Markdown 总结
  • 文件路径说明
  • 限制与 best-effort 说明
  • 下一步建议
核心思路
不是去硬绕过反爬虫，而是让系统像你的网页副驾驶一样，在你的真实浏览器会话里帮你完成重复劳动。
人工参与点
登录、滑块、验证码、Google 人类检查、2FA 这些都交给你，系统只负责在你处理完后继续执行。
当前边界
评论区和详情页属于 best-effort，平台个性化推荐会影响显示内容，但不影响这套流程作为真实会话辅助自动化使用。
